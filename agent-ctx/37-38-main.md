# Steps 37-38 — Borderless UI + Design Tokens + Missing UI States

## Summary
Applied borderless UI design pattern with CSS custom properties (design tokens) and added missing UI states (loading skeletons, empty states, error states, toast notifications) to the AutoFood admin dashboard.

## Changes Made

### Step 37: Borderless UI + Design Tokens

**globals.css** — Added design token CSS custom properties:
- `:root` (light): --color-bg, --color-bg-soft, --color-bg-muted, --color-bg-accent, --color-text, --color-text-secondary, --color-text-muted, --color-accent, --color-accent-hover, --color-success, --color-warning, --color-error, --color-border-subtle, --color-separator
- `.dark` (dark): matching dark mode values

**card.tsx** — Changed default card from `bg-card` to `bg-[var(--color-bg-soft)] border-none` for borderless appearance with background tint

**StatsCards.tsx** — Replaced `hover:shadow-sm` with `hover:bg-[var(--color-bg-muted)]` for subtle hover feedback without borders

**table.tsx** — Updated TableRow hover/selected states to use design tokens: `hover:bg-[var(--color-bg-muted)]`, `data-[state=selected]:bg-[var(--color-bg-accent)]`

**input.tsx** — Replaced full-border input with bottom-border pattern: `bg-[var(--color-bg-soft)] border-b border-b-[var(--color-separator)]`, focus highlights with `border-b-[var(--color-accent)]`

### Step 38: Missing UI States

**Created `src/components/ui/states.tsx`** — Reusable state components:
- `TableSkeleton({ rows, columns })` — Loading skeleton for data tables
- `CardSkeleton()` — Loading skeleton for card/dashboard views
- `EmptyState({ icon, title, description, action })` — Empty state with optional CTA
- `ErrorState({ title, message, onRetry })` — Error display with retry button

**OrdersTab.tsx** — Added loading state (TableSkeleton), empty state (EmptyState with "Create Order" button)

**StatisticsTab.tsx** — Added loading state (CardSkeleton grid), new `isLoading` prop

**ClientsTab.tsx** — Added loading state (TableSkeleton), empty state (EmptyState with "Add Client" button)

**BinTab.tsx** — Added empty states for both orders and clients tabs

**AdminDashboardPage.tsx** — Added toast notifications for handleCreateOrder (success + error)

**button.tsx** — Added `disabled:cursor-not-allowed` to base class

## Verification
- `npx tsc --noEmit` passes with 0 errors
- CSS custom properties defined in both :root and .dark
- states.tsx exports 4 reusable components
- 4+ tab components use loading/empty/error states
- 3+ mutation flows show toast notifications (order create, client create, courier create, finance transactions)
